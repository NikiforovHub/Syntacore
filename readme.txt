Описание алгоритма на MATLAB


Создаётся счётчик-вектор.
Алгоритм считывает из файла input.txt все векторы и перебирает все возможные комбинации векторов. 
Для каждого полученного вектора вычисляется вес, затем счётчик, соответствующий данному весу, 
увеличивается на единицу.
По полученным весам строится гистограмма весов.
Результат сохраняется в output.txt



Описание параллельного алгоритма на C++


Считываем в вектор-массив базисные векторы. Причём, каждый вектор будет представлен соответствующим десятичным числом. Например, для вектора 011 будет храниться десятичное число 3.
Сначала вычисляем одну векторную сумму. Для этого необходимо знать соответствующий код Грея. В нераспараллеленом варианте программы даже это не требуется, т.к. первый код Грея — последовательность, состоящая только из нулей.
Чтобы вычислить следующий вектор, находим номер изменившегося бита в коде Грея (строчка m = log2((~i & (i + 1)));).
Находим вес получившегося вектора. В векторе-массиве весов увеличиваем счётчик для соответствующего веса.
Таким образом вычисляем 2^K векторных сумм и весов.
Результат записываем в файл, который называется также, как исходный, только в папку outMy.

Все вычисления происходят на numThreads потоках. 
Изменяя переменную checkFiles можно установить будут ли сверяться полученные файлы с эталоном.
